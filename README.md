# resource-official-pc

本仓库用于归档与发布多个独立 HTML 页面。这些页面以静态资源形式存放，不针对任何特定网站或域名，仅供浏览与参考。

## 项目简介

resource-official-pc 是一个静态页面归档仓库，主要收集与维护若干独立 HTML 文件。每个页面均为自包含的 HTML 文档，可脱离后端环境直接运行。该仓库不涉及动态数据、用户交互或后端接口，适合作为资源展示、页面存档或原型演示使用。

## 目录说明

```
resource-official-pc/
├── index.html          # 入口页面（可选，用于导航或说明）
├── pages/              # 存放各归档页面的子目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/             # 静态资源文件（CSS、JS、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `index.html`：可选的主页，可汇总当前归档的所有页面链接。
- `pages/`：每个归档页面独立放置于此目录下，文件名建议使用有意义的英文或拼音。
- `assets/`：存放页面共用的样式、脚本和图片资源，避免重复引用。

## 页面归档说明

归档的 HTML 页面遵循以下规范：

- 每个页面均为独立完整的 HTML 文档，包含完整的 `<html>`, `<head>`, `<body>` 结构。
- 页面内不嵌入敏感信息、跟踪代码或外部依赖（除非必要且明确声明）。
- 页面样式与脚本建议尽量内联或引用仓库内 `assets/` 目录下的资源。
- 页面标题、描述等元信息应清晰准确，便于区分。

如需新增归档页面，请将 HTML 文件放入 `pages/` 目录，并确保文件命名不重复、不包含特殊字符。若页面需要额外资源，请一并放入 `assets/` 对应子目录中。

## 维护说明

- 本仓库由维护者定期更新，新增或修改页面时会通过 commit 记录变更。
- 欢迎通过 Issues 提出页面归档建议或报告问题，但请注意不涉及具体网站或域名。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/resource-official-pc.git
   ```
2. 直接在浏览器中打开 `pages/` 目录下的任意 HTML 文件即可查看。
3. 也可通过 GitHub Pages 或其他静态托管服务部署后在线访问。
